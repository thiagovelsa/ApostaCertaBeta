1. Coeficiente de Variacao (CV)

O CV mede a estabilidade dos dados historicos. Quanto maior o valor, mais imprevisivel e o comportamento do time. E calculado como o desvio padrao dividido pela media, expresso em valor absoluto.

Valor do CV Classificacao Interpretacao Tecnica Recomendacao
< 0.3 Muito Estavel Baixa dispersao dos dados. O time repete padroes com alta consistencia (ex: sempre 4-6 escanteios por jogo). APROVADO — Apostas seguras neste mercado.
0.3 — 0.6 Moderado Oscilacao controlada. O time varia, mas dentro de limites previsiveis. ANALISE NECESSARIA — Requer contexto adicional (clima, desfalques).
0.6 — 1.0 Instavel Alta variabilidade. Resultados imprevisiveis; a media nao reflete bem a realidade. EVITAR — Risco elevado de quebra de expectativa.
> 1.0 Caos / Incerteza Total Dispersao extrema. Exemplo: 0 escanteios num jogo, 14 no outro. A media matematica diz "7", mas e loteria pura. BLOQUEADO — Nao apostar neste mercado.
Exemplo Pratico

Time A: Media de 7 escanteios, CV = 1.2 (fez 0, 2, 14, 12, 8 nos ultimos 5 jogos)

Erro comum: Apostar Over 6.5 achando que "faz de media 7"

Realidade: 40% dos jogos ficam abaixo de 3 escanteios. E uma moeda viciada.

2. Tamanho da Amostra (n_used)

Representa quantos jogos foram usados para calcular as estatisticas. Afeta diretamente a confiabilidade dos modelos preditivos.

n_used Status Peso na Decisao Acao do Modelo
≥ 15 jogos Amostra Robusta 100% peso estatistico Usar dados puramente matematicos
10-14 jogos Amostra Aceitavel 80% dados / 20% contexto Pesquisar escalações recentes
5-9 jogos Amostra Fraca 40% dados / 60% noticias Priorizar analise tatica e pesquisa web
≤ 4 jogos Dados Insuficientes 10% dados / 90% contexto SUSPENDER ANALISE ESTATISTICA — Usar apenas dados de elenco e noticias recentes
Regra de Ouro

"n_used ≤ 3 significa que o calculo de media e matematicamente irrelevante para predicao. O desvio padrao nao tem significancia estatistica."

3. Estrategia: Casa vs. Fora

3.1 Quando priorizar last5_casa / last5_fora (especificos)

  • Times com mando de campo forte (ex: Fortaleza na Serie A, Galatasaray na Turquia)
  • Ligas com viagens longas (MLS, campeonatos asiaticos)
  • Times que jogam taticas diferentes em casa vs. fora (ex: time retranqueiro fora, ofensivo em casa)

3.2 Quando priorizar last10_corridos (gerais)

  • Classicos e rivais: O historico especifico de mando pode ser irrelevante devido a intensidade emocional
  • Finais de temporada: Motivacao supera estatistica de local
  • Times instaveis (CV > 0.8): A amostra geral dilui o "ruido" dos outliers
  • Estreantes em ligas: Sem historico suficiente em casa/especifico

3.3 Formula de Ponderacao Dinamica

# Logica de decisao para peso casa vs. geral SE (rivalidade == "alta" OU CV_geral > 0.8): peso_casa = 30% peso_geral = 70% SENAO SE (distancia_viagem > 1000km): peso_casa = 70% peso_geral = 30% SENAO: peso_casa = 50% peso_geral = 50%

4. Verificacao Anti-Alucinacao de Elenco

JSONs desatualizados podem conter jogadores que ja foram vendidos, machucados ou sairam do clube. Esta verificacao e obrigatoria antes de qualquer recomendacao de aposta.

4.1 Checklist de Validacao

  1. Pesquisar: "[Nome do Time] elenco atual [Ano]"
  2. Fontes confiaveis: Site oficial do clube, Transfermarkt, GE.globo, SofaScore
  3. Comparar: Cruzar jogadores citados no JSON com lista oficial atual

4.2 Tratamento de Divergencias

Situacao Status Acao
Jogador "chave" nao consta no elenco oficial DADO DESATUALIZADO Ignorar estatisticas relacionadas a este jogador
Jogador suspenso/lesionado confirmado na fonte oficial ALERTA ATIVO Ajustar analise ofensiva/defensiva considerando ausencia
Time com >3 alteracoes no elenco titular vs. JSON ELENCO RECONSTRUIDO Priorizar analise tatica sobre estatisticas historicas

4.3 Template de Consulta Obrigatoria

ANTES de sugerir aposta baseada em forca ofensiva:

"Elenco atual [Time] 2025/2026 Transfermarkt"

"Escalação provavel [Time] [Data do Jogo]"

"Desfalques confirmados [Time] globoesporte"

SE jogador citado no JSON NAO aparecer em nenhuma fonte atual:

→ Marcar: "[DADO DESATUALIZADO - IGNORAR]"

5. Integracao na Persona (RAG)

Quando o modelo receber o JSON de estatisticas, deve consultar este manual antes de gerar a analise final:

# PROTOCOLO DE ANALISE OBRIGATORIO: # 1. Verifique CV do mercado escolhido: SE CV > 1.0 → RECUSAR APOSTA, independente de valor esperado SE 0.6 < CV < 1.0 → Solicitar confirmacao de contexto favoravel # 2. Valide n_used: SE n < 5 → Ativar modo "Analise Qualitativa", suspender calculos Poisson SE n < 10 → Adicionar disclaimer "Dados limitados - alta variancia" # 3. Determine peso Casa/Fora: Classico? Use 70% last10_corridos Viagem longa? Use 70% last5_especifico CV alto (>0.8)? Sempre priorize amostra maior # 4. Execute Anti-Alucinacao: Validar top 3 jogadores ofensivos mencionados Confirmar lesoes/suspensoes nas ultimas 24h # 5. Output final deve incluir: - "Risco Estatistico: [Baixo/Medio/Alto] baseado em CV=X.XX" - "Confianca dos Dados: [Alta/Media/Baixa] (n=X jogos)" - "Validacao de Elenco: [Confirmada/Pendente/Desatualizada]"

6. Resumo Executivo para o Modelo

Indicador JSON Limite Critico Acao Automatica
CV > 1.0 Bloquear entrada no mercado
CV > 0.6 Alerta de cautela maxima
n_used < 5 Ignorar modelo matematico, usar so noticias
n_used < 10 Reduzir stake sugerida em 50%
Data do JSON > 30 dias Executar validacao de elenco obrigatoria
Principio Fundamental

"O JSON fornece a matematica. A Web Search fornece o tempo real. Este manual fornece a sabedoria para unir ambos."
